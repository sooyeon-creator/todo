# 할일+습관 트래커 -- Phase 분리 계획

> 한 번에 다 만들면 복잡해져서 품질이 떨어집니다.
> Phase별로 나눠서 각각 "진짜 동작하는 제품"을 만듭니다.

## Phase 1: MVP (2~3주)

### 목표
Google 로그인 후 할 일을 추가하고, 서브태스크로 쪼개서 Progress Bar로 진행도를 보고, 습관을 등록해서 스트릭을 확인할 수 있는 완전히 동작하는 웹앱.

### 기능
- [ ] Google 로그인 (Supabase Auth)
- [ ] Task 입력 (한 줄 + Enter로 즉시 생성)
- [ ] Task 목록 표시 (미완료/완료 구분)
- [ ] Task 완료 체크
- [ ] Task 클릭 → 디테일 캔버스(슬라이드 패널) 오픈
- [ ] 서브태스크 추가 (캔버스 내 체크박스 항목)
- [ ] 서브태스크 체크/해제
- [ ] Progress Bar (완료 서브태스크 수 / 전체)
- [ ] 습관 등록 (이름 + 반복 주기)
- [ ] 오늘의 습관 목록 표시
- [ ] 습관 완료 체크
- [ ] 스트릭 숫자 표시 (연속 달성 일수)

### 데이터
- Task, SubTask, Habit, HabitLog (04개 엔티티 모두 Phase 1에서 사용)

### 인증
- Supabase Auth + Google OAuth
- 로그인 안 한 사용자는 로그인 페이지로 리다이렉트

### "진짜 제품" 체크리스트
- [ ] 실제 Supabase DB 연결 (목업 데이터 X)
- [ ] 실제 Google 인증 (하드코딩된 사용자 X)
- [ ] Vercel에 실제 배포 (localhost X)
- [ ] 배포된 URL에서 다른 기기로 접속해서 데이터 동기화 확인

## Phase 2: 확장 (2주)

### 전제 조건
- Phase 1이 Vercel에 안정적으로 배포된 상태
- Task, Habit, HabitLog가 실제 데이터로 쌓이고 있음

### 목표
날짜 관리와 통계 시각화를 추가해 "언제까지 해야 하는지"와 "내가 얼마나 잘 하고 있는지"를 한눈에 파악.

### 기능
- [ ] Task에 마감일(due_date) 설정 UI
- [ ] 마감일 기준 정렬/필터
- [ ] 브라우저 푸시 알림 (마감일 하루 전, 습관 리마인더)
- [ ] 습관 히트맵 캘린더 (월간 달성 현황)
- [ ] 주간 달성률 차트 (Task + Habit 통합)

### 추가 데이터
- Task의 `due_date` 필드 활성화 (이미 스키마에 포함, 코드만 연결)
- Notification 설정 테이블 추가 가능성 있음

### 통합 테스트
- Phase 1 기능 (Task 추가, 서브태스크, 스트릭)이 Phase 2 이후에도 정상 동작하는지 확인

## Phase 3: 고도화 (추후)

### 전제 조건
- Phase 1 + 2가 안정적으로 운영 중
- 실제 사용하면서 불편한 점이 생겼을 때

### 목표
개인화 + 데이터 활용성 향상.

### 기능
- [ ] 태그/카테고리로 Task, Habit 분류
- [ ] 데이터 내보내기 (CSV)
- [ ] 다크/라이트 모드 테마 전환

### 주의사항
- 태그 기능은 새로운 테이블(Tag, TaskTag 등)이 필요 → 스키마 마이그레이션 주의
- CSV 내보내기는 민감한 개인 데이터를 포함하므로 보안 확인 필요
