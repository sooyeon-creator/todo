# 할일+습관 트래커 -- PRD (Product Requirements Document)

> 생성일: 2026-03-20
> 생성 도구: Show Me The PRD

## 1. 제품 개요

### 한 줄 요약
할 일 관리와 습관 트래킹을 하나의 웹앱에서 — 서브태스크 진행도와 스트릭으로 매일 성취감을 느끼는 개인용 생산성 도구.

### 해결하는 문제
- 할 일 앱과 습관 트래커를 따로 쓰느라 전환 비용이 발생함
- 큰 할 일을 쪼개서 관리하기 어렵고, 진행도를 한눈에 보기 힘듦
- 습관이 며칠째 이어지고 있는지 동기부여가 안 됨

### 핵심 가치
- **통합**: 1회성 할 일(Task) + 반복 습관(Habit)을 한 화면에서 관리
- **진행도 가시화**: 서브태스크 완료 수 / 전체 수를 Progress Bar로 즉시 확인
- **스트릭 동기부여**: 연속 달성 일수가 끊기기 싫어서 습관을 유지하게 됨

## 2. 사용자

### 주요 사용자
- **누구**: 개인 생산성을 높이고 싶은 성인 1명 (본인 사용)
- **상황**: PC 앞에 앉아 하루를 시작하거나 마무리할 때 브라우저에서 접속
- **목표**: 오늘 해야 할 일과 습관을 확인하고, 완료 표시를 하며 성취감을 느끼기

### 사용자 시나리오

**시나리오 A — 할 일 추가 및 서브태스크 관리**
1. 사용자가 입력창에 "프로젝트 보고서 작성"을 입력하고 Enter를 누른다
2. Task가 목록에 추가되면, 제목을 클릭해 디테일 캔버스를 연다
3. 캔버스에서 체크박스 항목("개요 작성", "데이터 정리", "결론 작성")을 추가한다
4. 하나씩 완료하면 Progress Bar가 1/3 → 2/3 → 3/3으로 채워진다

**시나리오 B — 습관 확인 및 스트릭 유지**
1. 사용자가 오후 7시에 앱을 열어 오늘 습관 목록을 확인한다
2. "러닝 30분" 옆에 스트릭 12일이 표시되어 있어 끊기 싫다는 생각이 든다
3. 운동 후 체크 표시 → 스트릭이 13일로 올라간다

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 — 할 일 완료

```
앱 열기 → 제목 입력 + Enter → Task 생성 → 제목 클릭 → 디테일 캔버스 → 서브태스크 추가 → 하나씩 체크 → Progress Bar 채워짐 → Task 완료
```

### 핵심 흐름 — 습관 달성

```
앱 열기 → 오늘의 습관 목록 확인 → 스트릭 확인 → 습관 완료 체크 → 스트릭 +1
```

### 상세 흐름
1. **첫 접속**: Google 로그인 → 메인 대시보드 (Task 목록 + Habit 목록)
2. **Task 생성**: 상단 입력창에 타이핑 → Enter → 목록 맨 위에 추가
3. **서브태스크 관리**: Task 제목 클릭 → 슬라이드 패널/모달 오픈 → "+" 버튼으로 서브태스크 추가 → 체크박스 클릭으로 완료
4. **습관 체크**: Habit 섹션에서 오늘 날짜 기준 미완료 목록 확인 → 체크 클릭 → 스트릭 숫자 업데이트

## 5. 성공 기준

- [ ] Task를 3초 안에 추가할 수 있다 (입력창 클릭 → 타이핑 → Enter)
- [ ] 에디터에서 `/[]` 입력 시 즉시 체크박스가 삽입된다
- [ ] Progress Bar가 체크박스 체크/해제 시 실시간으로 업데이트된다
- [ ] 체크박스 전부 완료 시 Task가 자동으로 완료 처리된다
- [ ] 습관 스트릭이 매일 정확하게 계산된다
- [ ] 구글 로그인 후 30초 안에 첫 Task를 만들 수 있다
- [ ] 배포된 URL에서 다른 기기에서도 데이터가 동기화된다

## 6. 안 만드는 것 (Out of Scope)

> 이 목록에 있는 건 Phase 1에서 만들지 않습니다.
> AI에게 코드를 시킬 때 이 목록을 함께 공유하세요.

- 팀 협업 기능 (공유, 댓글, 멤버 초대) — 이유: 개인용 앱이므로 불필요
- AI 우선순위 추천 — 이유: Phase 1 범위 초과
- 마감일 & 알림 — 이유: Phase 2에서 구현
- 주간/월간 통계 차트 — 이유: Phase 2에서 구현
- 모바일 앱 (iOS/Android) — 이유: 웹 브라우저 전용으로 시작
- 오프라인 모드 — 이유: Supabase 실시간 연동이 기본 전제
- 소셜 피드/공개 공유 — 이유: 개인 프라이버시 앱
